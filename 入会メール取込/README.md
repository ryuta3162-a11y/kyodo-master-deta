## 取りこぼし対策（旧版からの変更）

| 原因 | 対策 |
|------|------|
| Gmail 1回500件上限 | **月別**に検索＋100件ずつページ送り |
| メールアドレスで1人にまとめていた | **入会メール1通＝1行**（総数が正確に） |
| スレッド内の返信が二重 | 同一メールID・同一「宛先+分」は1行だけ |
| 名前の取りこぼし | お名前／氏名／先頭「〇〇様」を強化。未取得は備考に表示 |

## シート1の列

| 列 | 内容 |
|----|------|
| A | タイムスタンプ |
| B | 名前 |
| C | メールアドレス |
| D | 受付番号 |
| E | 備考（名前未取得など） |

**新しい入会が上**に並びます。

完了ダイアログに **総数**・**Gmailメッセージ数**・**名前未取得件数** が出ます。  
詳細は非表示シート **`_取込ログ`** に記録されます。

## 手順

1. `Code.gs` を Apps Script に全文貼り付け・保存
2. **入会メール取込 → Gmailから取込（全件）**
3. 総数と「名前未取得」を確認。未取得行は Gmail 本文と照合

## 赤い行の抽出

1. **シート1** で該当者の行を**赤く塗る**（1列でも行全体でも可）
2. **入会メール取込 → 「赤」シートへ抽出（赤い行）**
3. **「赤」** シートにその人だけ一覧化される（列構成はシート1と同じ）

スプレッドシート上の図形ボタンにも同じ関数 `kyodoExportRedMarkedRows` を割り当てられます。

## キャンペーン確認メール（HTML・一斉送信）

件名: `【重要】JOYFIT24経堂より：キャンペーン在籍期間に関するご確認`

| メニュー | 内容 |
|----------|------|
| キャンペーン確認メール（テスト1件・手入力） | 指定アドレスに1通だけ送信 |
| キャンペーン確認メール（**テストシート送信**） | シート名 **「テスト」** のメールアドレスのみ送信（件名に【テスト】） |
| キャンペーン確認メール（赤・一斉送信） | **赤**シートのメールアドレスへHTML送信（本番件名） |

### 「テスト」シートの作り方（送信前の予行練習用）

1. タブ名 **テスト** のシートを作る  
2. **A列だけ** に、テスト送信したいメールアドレスを1行1件で入力（ヘッダー行があっても @ が無ければスキップ）  
3. **テストシート送信** で、このA列の宛先だけに【テスト】付きで送信  

本番の **赤** シートとは完全に別です。

- 本文は **名前＋様** で個別化（HTML＋テキスト）
- 送信後、「赤」シートに **送信日時** 列を自動追加
- 既に送信日時がある行はスキップ（再送防止）
- 返信期限の文言は `Code.gs` の `KYODO_CAMPAIGN_REPLY_DEADLINE` で変更

**注意:** GASの `MailApp` は実行アカウントから送信されます。1日の送信上限に注意してください。

## 照合シート
