# リフォーマー拡張仕様（Phase 2）

## いま（Phase 1）の仕組み

| 質問 | 答え |
|------|------|
| 自動更新？ | **いいえ**（メニュー実行時のみ）。時間トリガーを付ければ自動化可 |
| Pilates と同じロジック？ | **会員数シートのスナップショットコピー**。HTMLダッシュボードそのものではない |
| 数値は正しい？ | **同期した瞬間**のチェック表「会員数」と同じ。その後チェック表だけ更新するとズレる |

```
チェック表「会員数」 ──コピー──→ アイデアプール「リフォーマー」
         ↑
   DETA更新・会員アクティブ率（Pilates本体）
```

**HTMLダッシュボード**（Index.html・Web画面）は別物。シートに映るのは「会員数」タブの表だけ。

---

## Phase 2 でやりたいこと

### A. 見た目（色・体裁）

- チェック表「会員数」の **セル色・フォント・罫線** をコピー（`copyTo` で書式込み）
- 完全な HTML 再現は不可 → スプレッドシート向けに **近い色** を GAS で付与

### B. 年度表の ⑥ 直下に ⑦⑧⑨ を統合（全インストラクター合計）

ダッシュボード（Pilates Reformer System）の **全員合計** を、既存の【2025/2026年度】表に追記（下部の別表は作らない）:

```
【2025年度】
  ① … ⑤
  ⑥全体利用率（既存）
  ⑦月間予約数（有効）  ← deta 集計（GAS 書込）
  ⑧平均稼働率          ← 数式（⑦ ÷ O列枠数×4）
  ⑨キャンセル率        ← 数式（P列 ÷ (⑦+P列)）
【2026年度】
  （同様）
```

| 行 | 内容 | 更新方法 |
|----|------|----------|
| ⑦ | 月間予約数 | GAS（deta から集計） |
| ⑧ | 平均稼働率 | スプレッドシート数式（手修正可） |
| ⑨ | キャンセル率 | スプレッドシート数式（手修正可） |
| O列 | 枠数（非表示） | GAS 書込。直すと ⑧ が変わる |
| P列 | キャンセル数（非表示） | GAS 書込。直すと ⑨ が変わる |

- インストラクター個人別は **不要**（全員合計のみ）
- データ元: `deta` シート（Pilates 本体と同じ集計ロジックを **アイデアプール側 GAS** で再現）

### 前提

**`pilates/元データ/Code.gs` に本体を貼る** → ダッシュボードの集計関数を読み取り、同じ結果になるよう実装する。
