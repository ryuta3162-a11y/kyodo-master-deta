# Pilates → アイデアプール（会員数）運用フロー

## 前提

| 項目 | 方針 |
|------|------|
| **pilates/Code.gs** | **機能は変えない**（予約取込・会員アクティブ率などそのまま） |
| **連携** | **アイデアプール側**の GAS で取りに行く |
| **コピー内容** | チェック表「会員数」＝アイデアプール「リフォーマー」（全く同じ） |

## なぜアイデアプール側で取るか

- リフォーマーブックの `onOpen`（DETA更新メニュー）を壊さない
- 連携だけ別ファイルで管理できる
- 司令塔（アイデアプール）から「最新にする」操作が自然

## 正しい操作の流れ

```
【経堂リフォーマーチェック表】
  1. DETA更新 → 会員アクティブ率の算出（会員数シートへ）  ← いつも通り

【経堂リニューアル　アイデアプール】
  2. 共同データ → リフォーマー会員数を更新  ← 新メニュー
```

**1のあとに2** で、アイデアプールの「リフォーマー」がチェック表と同じ内容になります。

## Apps Script の置き場所

| ブック | 追加するファイル | Git 正本 |
|--------|------------------|----------|
| リフォーマーチェック表 | `Code.gs` + `Index.html` のみ（**変更なし**） | pilates/ |
| **アイデアプール** | **`SyncFromReformer.gs` を追加** | [一元管理/SyncFromReformer.gs](../../一元管理/SyncFromReformer.gs) |

`pilates/SyncToIdeaPool.gs` は同じ処理の別置き（プッシュ用・参考）。通常はアイデアプール側を使う。

## 初回セットアップ

1. アイデアプール → 拡張機能 → Apps Script
2. **新規ファイル** `SyncFromReformer.gs` を作成
3. Git の [一元管理/SyncFromReformer.gs](../../一元管理/SyncFromReformer.gs) を貼り付け
4. 保存 → アイデアプールを開き直す
5. メニュー **「共同データ」** が出ることを確認
6. チェック表で会員数更新 → アイデアプールで「リフォーマー会員数を更新」
