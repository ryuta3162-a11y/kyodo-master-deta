# 元データ保管ルール

GAS・HTML の **壊れない正本** の扱い方です。

## 原則

| ルール | 内容 |
|--------|------|
| **1. 元データフォルダ** | 各プロジェクトに `元データ/` を置き、**ブックから取ったそのまま** を保存 |
| **2. 凍結** | `元データ/` は連携・実験用の改修を入れない |
| **3. 作業用コピー** | 親フォルダの `Code.gs` 等は作業用（元データのコピーから開始） |
| **4. 追加は別ファイル** | 連携は `SyncFromReformer.gs` のように **別 .gs** で足す |
| **5. 空にしない** | 中身があるファイルをプレースホルダーで上書きしない |
| **6. Git** | 元データを取り込んだら **コミット**（消えないようにする） |

## フォルダ例（Pilates）

```
pilates/
├── 元データ/          ← 正本（AirReserve そのまま）
│   ├── Code.gs
│   └── Index.html
├── Code.gs            ← 作業用（元データと同内容を維持）
├── Index.html
└── SyncToIdeaPool.gs  ← 連携だけ（本体に触らない）
```

## 新規ブックを追加するとき

1. `元データ/Code.gs` に Apps Script から貼る
2. 台帳 [スプレッドシート一覧.md](スプレッドシート一覧.md) に登録
3. コミット
